### rhize_orders (TABLE)
MySQL tenant ops table — one row per order line item. Columns include date (DATETIME), customerName, productName, qty, unitPrice, subtotal, status ('Completed'|'Pending'|'Cancelled'), openBalance, brand_id, tenantid. Use for: open orders, completed orders, customer revenue, sales by product. The user's OWN orders.
RESTRICT: Tenant predicate is auto-injected — never write tenantid yourself. Do NOT scope by brand_id (collides cross-tenant). Use ANSI single-quotes for strings.

### rhize_product_lots (TABLE)
MySQL tenant ops table — lot tracking. One row per (lotNumber, tenant). Columns: strain, lotNumber, thc, tac, terps, expirationDate, internalLotId, ccbLotNumber, tenantid. Use for: THC by strain, expiring lots, lot metadata.
RESTRICT: Tenant predicate is auto-injected.

### rhize_sales_actions — columns
- **priority** (INT): 1 = highest. ORDER BY priority ASC.

### rhize_product_lots — columns
- **thc** (DOUBLE): THC percentage. Already numeric (no cast needed). NULL-safe: WHERE thc IS NOT NULL when averaging.

### rhize_brands — columns
- **isRhize** (TINYINT): Flag: 1 = the user's OWN brand, 0 = competitor in their catalog. For 'my brand' / 'my own brand' questions filter WHERE isRhize = 1.
- **name** (VARCHAR): Brand name. Unique per (tenantid, name).

### rhize_orders — columns
- **subtotal** (DOUBLE): Line revenue (qty × unitPrice). SUM(subtotal) for total revenue. Combine with WHERE status = 'Completed' for realized revenue.

### complete_market_scrapper_dataset — columns
- **product_name** (TEXT): SKU display name. Use the base table (not the view) when product_name is needed.
  RESTRICT: Exclude blanks when ranking.
- **brand** (TEXT): Brand name (free-text). Use AND brand IS NOT NULL AND brand <> '' when grouping by brand to avoid blank buckets.
  RESTRICT: Exclude blanks for any GROUP BY brand.
