### rhize_brands (TABLE)
MySQL tenant brand registry. Columns: id (cuid), name, isRhize (tinyint — 1 = the user's OWN brand, 0 = competitor brand in their catalog), createdAt, tenantid. Use for: the tenant's own brand (isRhize=1), brand catalog list. Unique on (tenantid, name).
RESTRICT: Tenant predicate is auto-injected. For the user's OWN brand identity filter `WHERE isRhize = 1`.

### complete_market_scrapper_dataset (TABLE)
Base fact table: one row per (date, state, company, product, unit). Scraped market/competitor listings. RLS-filtered by state. Use this table only for SKU-level detail (product_name, price, THC, flag, days_on_shelf). For brand/company/category aggregates, use chatbot_mv_market_daily — it's ~480x smaller per day.
RESTRICT: Never filter by state manually — RLS does it. Never write to this table. SCOPE: market/competitor data only, NOT own-business data.

### complete_market_scrapper_dataset — columns
- **brand** (TEXT): Brand name (free-text). Use AND brand IS NOT NULL AND brand <> '' when grouping by brand to avoid blank buckets.
  RESTRICT: Exclude blanks for any GROUP BY brand.
- **company** (TEXT): Retailer/dispensary name (free-text). Same blank-filter rule as brand.
  RESTRICT: Exclude blanks for any GROUP BY company.
- **revenue** (DOUBLE PRECISION): price_num * |change|. Inferred market sell-through, NOT POS revenue.
  RESTRICT: Default ranking metric for top/best/biggest.

### chatbot_mv_market_daily — columns
- **brand** (TEXT): Brand grouping key. Exclude blanks: brand IS NOT NULL AND brand <> ''.
  RESTRICT: Exclude blanks for any GROUP BY brand.
- **sku_count** (INTEGER): Distinct SKUs per grouping. Use SUM(sku_count) when grouping by brand for 'most products listed'.
- **revenue** (NUMERIC): Daily sum of base-table revenue per (brand, company, category, state).
  RESTRICT: Default ranking metric.

### rhize_brands — columns
- **name** (VARCHAR): Brand name. Unique per (tenantid, name).
- **isRhize** (TINYINT): Flag: 1 = the user's OWN brand, 0 = competitor in their catalog. For 'my brand' / 'my own brand' questions filter WHERE isRhize = 1.
